1. Website Overview

Boho Cafe & Lounge is a premium single-page restaurant website with a fully integrated digital reservation and table management system. The website serves two audiences:

👤
For Customers
Browse the cafe, view menu, explore gallery, check events, make table reservations with live availability, and track booking status — all from one beautiful interface.
👑
For Owner / Admin
Manage all reservations, approve/reject with table assignment, view daily timeline, search/filter bookings, receive email alerts, and track cafe occupancy in real-time.

3. Reservation System — Customer Flow

Key Feature: Customers can book specific tables with live availability checking, preventing double-bookings and conflicts. The system supports individual table bookings, full cafe reservations, and event bookings.

3.1 Booking Types

TypeDescriptionUse Case
🪑 Table BookingReserve a specific table (1–10) for a time rangeRegular dining, date nights, small groups
🏠 Full CafeReserve the entire cafe for exclusive usePrivate parties, corporate events
🎉 Event BookingReserve for a special event or celebrationBirthdays, anniversaries, launches

3.3 Live Table Availability

When a customer selects a date, start time, and end time, the system automatically checks the database and displays:

  • ✅ Available Tables — Shown in green, clickable to select
  • ❌ Unavailable Tables — Shown in red with reason ("Reserved by Guest")
  • ⏳ Pending Tables — Tables with pending requests (shown as warning)
  • Full Cafe Block — If the entire cafe is reserved, all tables show as blocked

3.4 Customer Reservation Flow

Customer Opens Website → Clicks "Book a Table" → Selects Booking Type → Picks Date & Time Range

System Checks Availability → Selects Available Table → Fills Details & Submits

System Validates & Saves → Emails Sent (Owner + Customer) → Gets Reservation ID

3.5 After Submission

  • Success screen with animated confirmation
  • Unique Reservation ID displayed with copy-to-clipboard button
  • "Track Status" button linking to status page
  • "Book Another" button to reset the form

4. Reservation System — Owner/Admin Flow

4.1 How the Owner Gets Notified

When a customer submits a reservation, the owner immediately receives an email with:

  • Customer name, phone, email
  • Booking type (Table / Full Cafe / Event)
  • Date, time range, guest count, occasion
  • Special requests (if any)
  • One-click ✅ APPROVE and ❌ REJECT buttons

4.2 One-Click Email Approval

Email Arrives → Owner Clicks APPROVE → Status → CONFIRMED → Customer Gets Confirmation Email

Email Arrives → Owner Clicks REJECT → Status → REJECTED → Customer Gets Rejection Email

4.3 Admin Dashboard Approval (with Table Assignment)

Opens Admin Dashboard → Clicks "Approve" → Table Selection Modal Opens

Shows Available/Occupied Tables → Selects Table Number → Confirms & Sends Email

5.2 Conflict Detection Rules

  • Table Overlap: If Table 3 is confirmed for 2:00 PM – 5:00 PM, no one else can book Table 3 during any overlapping time range.
  • Full Cafe Block: If the full cafe is confirmed for 6:00 PM – 10:00 PM, ALL tables are blocked during that time.
  • Full Cafe Request vs Existing Tables: If any table has a confirmed reservation, full cafe booking is rejected for that time.
  • Time Overlap Logic: Start1 < End2 AND Start2 < End1 (standard interval overlap check).

5.3 Operating Hours

The system supports 12 one-hour time slots: 12:00 PM to 11:00 PM. Customers can book any range within these hours (e.g., 2:00 PM – 6:00 PM for a 4-hour reservation).

6. Email Notification System

The system sends 4 types of automated emails via Resend API:

#Email TypeRecipientTriggerContents
1 New Reservation Alert Owner Customer submits reservation Full details + APPROVE/REJECT buttons
2 Booking Received Customer Customer submits reservation Reservation details + ID + Track Status link
3 Booking Confirmed Customer Owner approves reservation Confirmation + Assigned table number + Date/Time
4 Booking Rejected Customer Owner rejects reservation Rejection notice + WhatsApp contact link

All emails feature the Boho Cafe branding with dark theme, gold accents, and responsive HTML design.

7. Admin Dashboard Features

Access: /admin — Password protected with session-based authentication.

7.1 Dashboard Statistics

Four live counters at the top showing:

📊 Total Reservations
All-time count of all reservation requests.
⏳ Pending
Reservations awaiting owner approval.
✅ Confirmed
Approved reservations with assigned tables.
❌ Rejected
Declined reservation requests.

7.2 List View

  • Filter Tabs: All / Pending / Confirmed / Rejected
  • Search Bar: Search by customer name, email, or phone
  • Reservation Cards showing: Name, status badge, booking type, phone, email, date, time range, guest count, occasion, reservation ID, special requests, submission timestamp
  • Approve Button: Opens table assignment modal (for pending reservations)
  • Reject Button: Immediately rejects with email notification
  • Assign Table Button: For confirmed reservations without a table assignment

7.3 Timeline View

  • 10 × 12 Grid: 10 tables (rows) × 12 time slots (columns: 12 PM – 11 PM)
  • Date Navigation: Previous/Next day buttons + date picker
  • Color-Coded Cells:
    • Green — Confirmed reservation (shows customer first name)
    • Amber — Pending reservation
    • Gray — Free / Available
  • Hover Tooltip: Shows full customer name, guest count, time range, status
  • Refresh Button: Manually reload timeline data
  • Unassigned Section: Lists confirmed reservations without table assignments

7.4 Table Assignment Modal

  • Shows customer name, guest count, date, and time range
  • 10 table buttons in a grid layout
  • Available tables — Clickable, shows "Free"
  • Occupied tables — Disabled, red, shows "Busy" (based on time overlap)
  • Error message if attempting to assign a busy table
  • Confirm button to finalize assignment

8. Reservation Status Tracking

Access: /reservation/status — Public page, no login required.

8.1 How Customers Track Their Booking

  • Enter Reservation ID + Email address
  • System verifies and displays current status
  • Status displayed with visual indicators:
    • ⏳ PENDING — Awaiting owner confirmation
    • ✅ CONFIRMED — Booking approved, table assigned
    • ❌ REJECTED — Booking declined
  • Full reservation details shown: date, time, guests, table number, occasion
  • Direct link provided in confirmation email (auto-fills ID and email)
